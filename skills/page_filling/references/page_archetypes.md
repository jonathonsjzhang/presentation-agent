# 页型与证据范式

`page_type` 必须取 schema 枚举。页型决定证据结构，不决定固定模板。

| page_type | 必备内容 | 推荐构图 |
|---|---|---|
| `methodology` | 数据来源、样本/时间、指标定义、检验方式、局限 | 定义 + 样本 + 可信性检验 |
| `executive_summary` | 顶层判断、2-4 个子发现、每个发现的核心数据 | 结论树或带证据的摘要 |
| `deep_dive` | 主证据、至少一种对比/拆解、机制或解释、边界 | 一个分析构图，可含多个联动视图 |
| `comparison` | entities × dimensions × cells、差异判断、来源 | 表格、热力图、散点或多面板 |
| `framework` | 轴/分类定义、实体位置、判断依据、局限 | 2×2、价值链或分类框架 |
| `recommendation` | what、why、证据强度、待验证项、caveat | 优先级序列，不编目标值/owner |
| `appendix` | 补充数据、口径和来源 | 明细表或补充图 |
| `cover` | 标题、主题、日期/范围 | 极简封面 |
| `section_divider` | 章节问题或章节判断 | 极简过渡 |
| `closing` | 回扣结论、待决策事项 | 决策清单 |

## 密度判断

- Deep-dive 的“一主视觉”指一个完整分析构图，不等于只能放一张图；矩阵、拆解图、quote 和 caveat 可以组成联动视图。
- 对比页优先让读者沿同一维度横向扫实体，不要求读者跨三段 prose 自行比较。
- 方法论页不强求业务主数值，但必须让读者判断研究是否可信。
- 推荐页只使用已有证据能支撑的强度；没有来源的目标值、owner 和 deadline 写“待业务方确认”。
