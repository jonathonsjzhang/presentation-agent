# 信息充分性

信息密度不是堆 bullet，而是让一页完成「观点 → 证据 → 对比/拆解 → 边界 → 含义」。

## 三层证据

1. **主证据（必须）**：直接支撑 page takeaway 的具体事实或量化数据。
2. **辅助证据（数据型页至少一种）**：基线、同行、历史、分群、渠道、趋势或机制解释。
3. **边界信息（必须）**：指标定义、时间窗、样本/对象范围、显著性状态、替代解释或局限。

## 强度校准

| 缺口 | 处理 |
|---|---|
| 主证据不具体 | `claim_strength=hypothesis`，请求补数 |
| 无对比基线 | 不写“领先、提升、更优” |
| 无机制证据 | 不写因果，只写相关或观察 |
| 无边界信息 | 明示“据此不能回答什么” |
| 上游一页承载多个独立结论 | 发出 `storyline_change_requests`，建议拆页 |

## 经核验的结构示例

附件人工稿把“强留存”定义为**最近一周主用该产品**，不是 7 日或 30 日留存。其纯白对比页展示：

- 豆包：产品纯白 60%，非产品纯白 50%，差值 +10pp；
- DS：产品纯白 45%，非产品纯白 27%，差值 +18pp；
- 元宝：产品纯白 26%，非产品纯白 12%，差值 +14pp；
- 边界：纯白定义为使用当前 AI 产品前未使用过其他同类产品。

这个示例只说明证据结构。生成其他任务时必须从当前输入取数并绑定 `source_ref`，不得复制示例数字。
