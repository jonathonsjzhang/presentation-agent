# Page Filling Gotchas

以下规则来自真实失败案例。命中时修复内容，不要只补字段。

1. **分析页退化为摘要页**
   错：只有 KPI 和结论。
   对：保留主证据、辅助拆解和边界，展示“凭什么”。

2. **把 caveat 改成行动建议**
   错：“学校/线下带来更多纯白用户，但留存提升不显著”被改成“线下是最优渠道”。
   对：区分获客量和留存效果，不显著结论必须可见。

3. **把探索方向改成否定结论**
   错：“社交裂变尚无成熟样板、值得探索”被改成“短期不做”。
   对：表达为“可探索、需验证”，除非输入明确要求停止。

4. **编造行动感**
   错：无来源地生成“本季度 +5pp、某 TL owner、两周完成”。
   对：只写验证方向或待业务确认。

5. **标题强于证据**
   错：相关写成因果，局部写成全局，机会写成立即投入。
   对：用 `claim_strength` 校准措辞。

6. **visual plan 只有图表名**
   错：“做柱状图”。
   对：写清问题、读图结论、必需数据、证据来源和局限。

7. **对比退化为并列 prose**
   错：依次描述三个产品。
   对：使用 `comparison_matrix`，按同一维度横向比较。

8. **用户原声被当作量化证据**
   错：几位用户的说法证明总体留存更高。
   对：quote 只解释机制、举例或提供反例。

9. **关键证据留在 JSON 深层**
   错：quant 很完整，但 handoff 没要求上屏。
   对：加入 `must_render_evidence` 和 `on_screen_numbers`。

10. **为了守页数静默删证据**
    错：把方法论、关键矩阵或反方直接删除。
    对：发出 `storyline_change_requests`，由 Manager 决定拆页、进附录或调整页数。
