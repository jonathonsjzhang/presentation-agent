# Professional Consulting Style Presentation Guide (v2.0)

> 本指南提供专业咨询风格（McKinsey/BCG 水准）的 PPT 呈现形式参考。
> 版本：v2.0 | 更新：2026-07-01
> 当 `output_format=ppt` 且目标受众为高管/董事会/战略决策层时，建议遵循本指南。
> 若用户提供了特定的 PPT 模板，优先适配用户模板。
> 示例参考：`examples/retention-case1/`（AI 产品留存分析案例）

## 0. Execution Summary — 论点树结构（P0）

封面后第一页使用**分层论点树结构**，替代并列卡片：

```
核心发现

发现1：纯白用户价值高但新增难度大（主论点）
  └─ 发现1.1：打"功能纯白"是更可行的路径（子发现）
  └─ 发现1.2：在非纯白用户中，迁移DS用户更易（子发现）
发现2：功能满意度驱动留存差异（主论点）
  └─ 发现2.1：可靠性是基础门槛
  └─ 发现2.2：深度思考是差异化焦点
发现3：元宝可尝试创新召回手段（主论点）
```

### 设计规范
- 主论点：14pt Bold，品牌色
- 子发现：11pt Regular，主文本色，缩进 1 个制表位
- 编号连续（发现1 → 发现1.1 → 发现1.2 → 发现2 → ...）
- 不配图表，纯论点结构

## 1. 导航标签栏（推荐）

每页顶部显示分析维度标签，帮助观众理解当前所处的分析视角。

### 结构
```
[维度1] [维度2] [维度3] [维度4] [维度5]
  ▲                                             [标题区]
  当前激活维度
┌──────────────────────────────────────────────┐
│                                              │
│              正文区（~70% 页面）               │
│                                              │
└──────────────────────────────────────────────┘
[来源脚注栏]
```

### 设计规范
- 标签数量：3-5 个维度
- 激活状态：品牌色填充 + 白色文字
- 非激活状态：浅色底 + 灰色文字
- 标签宽度：动态计算，min=1.2in，padding=0.3in

## 2. 发现呈现多样化（推荐）

根据内容特性选择不同的发现呈现方式，**禁止所有页面使用同一格式**：

### 方式 A：作为页标题（数据分析页首选）
```
发现1：纯白用户强留存显著高于非纯白，对元宝和DS的拉动更大
   ┌─────────────────────────────────┐
   │        主视觉（图表/表格）       │
   │  + 左侧数据标注 / pp 标注       │
   └─────────────────────────────────┘
```
发现结论作为行动标题的补充，直接嵌入标题区。

### 方式 B：作为侧边分析面板（默认推荐）
```
┌──────────────────────┬─────────────────┐
│                      │  发现            │
│    主视觉区域         │  ├─ 要点 1      │
│    （~75% 正文区）    │  ├─ 要点 2      │
│                      │  ├─ 要点 3      │
│                      │  └─ 要点 4      │
└──────────────────────┴─────────────────┘
```

### 方式 C：作为图表内数据标注（分析页推荐）
直接在图表中使用 pp 标注/红框/编号圆圈突出关键发现，无需独立面板：
```
柱状图：  豆包 54% ── +10pp标注
          DS   34% ── +15pp标注   <-- 红框高亮
          元宝 19%
```

### 方式 D：作为引文+分析联动框（定性分析页推荐）
```
┌──────────────────────────┬─────────────────────┐
│    主分析内容             │  "用户原声引用"     │
│                          │   ── 分析：         │
│                          │   引文支持的结论     │
└──────────────────────────┴─────────────────────┘
```

### 选用规则
- 数据分析页优先选 A（标题融入）或 B（侧边面板）
- 有显著数据差异的页优先选 C（图表标注）
- 有用户原声/定性数据的页优先选 D（引文框）
- 复杂多维度页选 B（侧边面板）

## 3. 数据强调

### pp 标注
在图表中直接用 "+Xpp" 或 "-Xpp" 标注差异值：
- 正值：绿色或品牌色
- 负值：红色
- 格式：`+10pp`，字体 11pt Bold

### 红框/编号圆圈高亮
标出图表中最关键的数据点：
- 红框：线宽 1.5-2pt，轻微圆角
- 编号圆圈：直径 0.3-0.4in，背景色为品牌色，白色数字

## 3a. 散点图（新增，数据分析首选）

当需要同时展示两个维度的数据关系时（如"强留存提升(pp) vs 强留存率(%)"），优先使用散点图。

### 适用场景
- 展示多个产品的"投入 vs 产出"关系
- 展示"幅度"和"率"两个维度的联合分布
- 发现数据点的聚类/离群模式
- 对比多个实体的双维度表现

### 结构
```
强留存提升(pp)
  ↑
  │    ● DS
  │          ● 豆包
  │    ● 元宝
  └──────────────────────→ 强留存率(%)
```

### 设计规范
- 横轴：基线指标（如强留存率 %）
- 纵轴：变化指标（如强留存提升 pp）
- 每个点代表一个产品/维度，用品牌色区分
- 可添加趋势线或象限分割线辅助解读
- 关键数据点加数据标签

## 4. 脚注和注释系统

每页底部显示：

| 类型 | 示例 | 必要性 |
|------|------|--------|
| 数据来源 | "样本 N=4,174，来自问卷调查" | 🔴 必须 |
| 关键定义 | "强留存率定义为过去一周主用该产品的用户占比" | 🟡 推荐 |
| 计算方法 | "留存提升 pp = 纯白留存率 - 非纯白留存率" | 🟡 推荐 |
| 符号说明 | "* 表示 p<0.05, ** 表示 p<0.01" | 🟢 可选 |

## 5. 信息密度模板

### 方案 A：左图右文（默认首选）
```
┌──────────────────────┬─────────────────┐
│                      │  发现/注解       │
│    图表/表格          │  ├─ 要点 ...    │
│    （~55% 正文区）    │  └─ 要点 ...    │
│                      │                 │
└──────────────────────┴─────────────────┘
```
适用：数据分析页（~60% 的场景）

### 方案 B：三栏布局
```
┌─────────┬─────────┬──────────────────┐
│ 栏 1    │ 栏 2    │  发现/注解       │
│ 图表/   │ 图表/   │  ├─ 要点 ...     │
│ 表格    │ 表格    │  └─ 要点 ...     │
└─────────┴─────────┴──────────────────┘
```
适用：多维对比分析（~20% 的场景）

### 方案 C：上下分层
```
┌──────────────────────────────────────┐
│               上方（图表/方法论）       │
├──────────────────────────────────────┤
│           下方（结论/总结）            │
└──────────────────────────────────────┘
```
适用：方法论/总结页（~15% 的场景）

### 方案 D：纯文字结构化（仅限少数场景）
```
┌──────────────────────────────────────┐
│           主标题                      │
│  ┌────────┐  ┌────────┐             │
│  │ 要点 1 │  │ 要点 2 │             │
│  └────────┘  └────────┘             │
│  ┌────────┐  ┌────────┐             │
│  │ 要点 3 │  │ 要点 4 │             │
│  └────────┘  └────────┘             │
└──────────────────────────────────────┘
```
适用：执行摘要/行动建议（~5% 的场景），必须用形状/卡片/结构化布局呈现，不得仅用纯文字段落

## 6. 通用配色方案

### 默认配色
| 元素 | 色值 | 用途 |
|------|------|------|
| 品牌主色 | `#0052D9` | 导航激活、强调元素 |
| 产品 A | `#22C55E` | 数据点、条形图 |
| 产品 B | `#2563EB` | 数据点、条形图 |
| 产品 C | `#93C5FD` | 数据点、条形图 |
| 成功/正向 | `#22C55E` | pp 标注正值 |
| 警告/负向 | `#EF4444` | pp 标注负值、高亮 |
| 面板背景 | `#F5F5F5` | 注解/发现区域背景 |
| 主文本 | `#333333` | 标题、正文 |
| 辅助文本 | `#666666` | 辅助文字、非激活标签 |

用户可通过 `style_tokens.color` 自定义配色方案。不同品牌使用不同颜色区分。

## 7. MckEngine API 参数格式速查

MckEngine 是 PPT renderer 的核心引擎，其 API 参数格式如下：

### 颜色参数处理
```python
BRAND_COLORS = {
    'primary': '#0052D9',
    'product_a': '#22C55E',
    'product_b': '#2563EB',
}
def hex_to_rgb(hex_color):
    hex_color = hex_color.lstrip('#')
    return RGBColor(int(hex_color[0:2], 16), int(hex_color[2:4], 16), int(hex_color[4:6], 16))
def get_color(key): return hex_to_rgb(BRAND_COLORS.get(key, '#0052D9'))
```

### 常用布局方法参数格式

| 方法 | 参数格式 | 常见错误 |
|------|---------|---------|
| `executive_summary` | items=`[(整数, "标题", "描述"), ...]` | ❌ 使用字典列表 |
| `metric_cards` | cards=`[("字母", "标题", "描述"), ...]` | ❌ 使用字典列表 |
| `funnel` | stages=`[("阶段名", "数值标签", 小数), ...]` | ❌ 使用字典列表 |
| `grouped_bar` | series=`[("系列名", "color_str"), ...]` | ❌ 使用字符串列表；不支持 `bottom_bar` |
| `horizontal_bar` | items=`[("项名", 整数百分比, "color_str"), ...]` | ❌ 使用字典列表 |
| `action_items` | actions=`[("标题", "时间线", "描述", "负责人"), ...]` | ❌ 使用字典列表 |
| `four_column` | items=`[("标题", "数值", "描述"), ...]` | ❌ 使用字典列表 |

> **注意**：以上格式基于 `mck_ppt.engine.MckEngine` 源码。如遇 API 不一致，以 `engine.py` 中的实际方法签名为准。

### 常见错误排查

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| `unexpected keyword argument 'bottom_bar'` | 方法不支持此参数 | 删除 `bottom_bar` |
| `too many values to unpack (expected 2)` | series 格式错误 | 改为 `(name, color_str)` 元组 |
| `assigned value must be type RGBColor` | 颜色是字符串 | 添加 hex_to_rgb 转换 |
| `not enough values to unpack` | 元组长度不对 | 检查元组元素个数 |

## 8. QA 质量检查清单

渲染后建议运行以下检查：

### 内容检查
- [ ] 所有图表数据有来源追溯
- [ ] 无模拟数据、占位数据
- [ ] 无 placeholder 文本（xxxx, lorem, ipsum, TODO）
- [ ] 每页有至少一个视觉元素

### 布局检查
- [ ] 无文本溢出（标题 ≤ 40 字，insight ≤ 5 条）
- [ ] 无形状重叠（间距 ≥ 0.1 英寸）
- [ ] 所有元素在安全区内（距边界 ≥ 0.5 英寸）

### 格式检查
- [ ] 封面后第一页是 Execution Summary
- [ ] 没有目录页或章节分隔页
- [ ] 渲染真实完成（render_result.rendered=true）
