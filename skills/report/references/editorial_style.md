# Strategy report editorial style

本参考只用于 Report 的正式写稿和最终编辑。它控制表达，不改变已批准的论点、顺序、证据强度和停止点。

## 管理者语言

- 在 Executive Summary 和章节开头先给判断，再展开范围、依据和边界；不要求每个段落机械重复结论。
- 做有方向的判断，同时保留必要边界。使用“主要、相对、整体、短期、初步判断、更适合”等准确限定，不用含糊措辞逃避判断。
- 保持专业、克制和自信。使用清楚的主语、具体业务对象和证据允许的动词；不要通过“驱动、锁定、重塑”等强动词把相关性升级为因果。
- 熟悉的行业术语在提高精度时可以保留；抽象术语若不能让判断更准确，应改成可理解的业务语言。

## 让证据进入句子

- 数字、案例和原话应紧邻它们支持的判断，并说明它们证明的是趋势、比较还是机制线索。
- 比较之后说明差异意味着什么；不要停在“A 高于 B”或“各有特点”。
- 上游提供且会影响判断时，写明样本、定义、时间窗、单位、可比性或方法限制；普通方法信息不要压过主结论。
- 不把相关性写成确定因果，不把个案写成总体规律，也不通过更强语气弥补证据不足。

把证据限制写成读者需要的判断边界，而不是 Worker 的审稿记录：

- 避免：“本批材料支持 A，但现有证据不能支持 B。”
- 改为：“当前可以观察到 A；B 的具体幅度、因果关系或适用阈值仍需通过相应数据或试点验证。”
- 可以说明“基于访谈、样本有限或缺少内部数据”，但这类信息必须直接校准结论，不能用于展示分析过程、置信度评审或任务完成情况。

## 长文节奏

- Executive Summary 只保留核心回答、少量关键依据、任务要求的决策含义和会改变判断的主要边界；它不是背景介绍或章节目录。
- 正文按 Storyline 完成完整证据链。判断、证据、解释和含义可以跨段自然展开，不要求每段套同一模板。
- 使用短段落和有判断的描述性小标题，让章节自然推进，而不是把 Storyline brief 逐项扩写。
- 同一结论只有在增加新证据、解释或含义时才重复。摘要、章节开头、章节结尾和全文收束不做同义复述。

## 句子习惯

- 优先使用短而有信息量的句子；一句话包含两个分析转折时拆开。
- 冒号可用于连接判断与解释，括号可承载不应打断主句的短定义或方法说明，但不为形成固定风格而滥用。
- 只有类别真正可比时才使用排比或平行结构，不为凑“三点”制造第三项。
- 只有 Storyline 已批准建议时才写建议；在证据支持范围内说明优先级、对象、作用机制和适用条件或时点。

## 删除常见 AI 腔

- 避免反复使用“不是……而是……”，以及先制造虚假对立再给结论的句式。
- 删除没有具体对象或证据的“重塑格局、构筑壁垒、开启新范式、全面赋能”等战略套话。
- 不用多句近义表达反复包装同一个观点，不强行三段对称，不滥用引号、口号式短语和装饰性过渡。
- 不解释听众已经知道且不改变判断的常识，不用背景教育填充篇幅。

## 最后一次编辑

通读时只追问：摘要能否直接回答问题；关键判断是否有明确依据；比较是否产生了 so what；删除某句话后是否不损失判断、证据、推导或必要边界。能无损删除的内容就删除，不把这些问题扩张成新的硬检查清单。
