# Analysis Skill 修改方案

## 一、修改目标

本轮先明确 Analysis 在四阶段工作流中的定位和工作方法，不讨论 schema、字段、上下文投影等实现接口，也不直接改写现有 Analysis skill。

四个 Worker 模拟人类分析师生产正式汇报材料的过程：

```text
参考材料
  → Analysis：发现、验证并收敛观点
  → Storyline：选择主线并组织论证
  → Report：沿故事线完成正式写作
  → Format：转译为目标载体
```

Analysis 的核心定位是：

> 通读参考材料，发散可能的发现、解释与假设，经过比较、追问和证据检验后，收敛成一组重要且站得住的发现与观点，为 Storyline 提供充足、清晰、保留边界的内容原料。

更简洁地说：

> Analysis 负责把材料想明白；Storyline 负责把想明白的观点讲成一个故事。

Analysis 不是材料摘要，也不是自由联想式 brainstorm，更不是提前写 Storyline。它是一个“先发散、再验证、后收敛”的观点生产过程。

## 二、新版 Skill 的现状判断

新版 Analysis skill 相比旧版已经增加了几个重要能力：

1. 从平铺材料转向先分解问题、再形成 findings。
2. 强调从现象继续追问用户、行为、机制和能力，不止步于第一层观察。
3. 引入可行动性、空间、收益、可行性和替代路径等筛选标准。
4. 要求 viewpoint candidates 说明互斥和依赖关系，开始考虑下游 Storyline 如何使用。
5. 继续保留证据追溯、反证、异常值、替代解释和置信度等分析底线。

这些方向与“观点发现与收敛”的定位基本一致，应当保留。

当前主要问题不是分析力度不足，而是发散—收敛过程仍不够清楚，并且部分规则开始越过 Analysis 与 Storyline 的边界。

### 1. 缺少显式的观点发散阶段

当前 workflow 从“理解任务、搭框架”直接进入“形成 Findings”。这容易让模型过早服从第一个分析框架，只寻找能够填入框架的材料，遗漏异常信号、跨来源联系和框架之外的发现。

人类分析师通常会先广泛浏览材料，记录可能有意思的现象、矛盾、解释和假设，再对候选内容进行验证与筛选。新版 skill 应把这一过程明确写出来。

### 2. “每个 finding 必须有张力”过于绝对

张力是好观点和核心主张的重要特征，但不是每条 finding 都必须同时包含机会与限制。部分基础 finding 的价值在于准确说明事实、比较、结构或机制。

如果强迫每条 finding 都写出张力，模型可能为了句式完整人为制造转折，把简单而重要的事实写得过度戏剧化。更合理的要求是：

- 原子发现必须清楚、具体、可检验；
- 综合观点应体现反常识、矛盾、边界或决策张力；
- 关键限制可以由 counter evidence、alternative explanation 或 caveat 承担，不必全部塞进 statement。

### 3. “金字塔与 MECE”容易与 Storyline 重叠

Analysis 确实需要结构化分析，但它使用结构的目的应是扩大和校验搜索范围，而不是提前形成汇报金字塔。

Analysis 需要的是“问题分解”和“分析框架”：

- 问题由哪些因素构成；
- 哪些比较、分群和时间维度值得检查；
- 哪些机制可能解释现象；
- 哪些证据能够支持或推翻判断。

Storyline 才需要“主张金字塔”和“叙事层级”：

- 哪个观点成为 apex；
- 哪三到五条信息支撑它；
- 受众先听什么、后听什么；
- 哪些内容进入正文、附录或被舍弃。

因此，新版中“主发现 → 子发现 → 支撑发现”的内容关系可以保留，但应避免使用容易指向汇报结构的金字塔语言，也不要求 Analysis 提前形成唯一层级。

### 4. 筛选标准有过早淘汰内容的风险

“可行动性优先”和“空间 × 收益 × 可行性”适合筛选最终观点，但不应成为发现阶段的唯一入口。

有些发现暂时不可行动，却可能：

- 改变对问题的定义；
- 推翻原有认知；
- 解释为什么现有策略失效；
- 构成重要风险或边界；
- 为其他观点提供机制支撑。

建议采用两轮筛选：

1. **发现阶段看信息价值**：重要、异常、反常识、可解释、可证伪。
2. **收敛阶段看汇报价值**：决策相关性、证据强度、影响空间和可行动性。

### 5. “追问到根因”需要明确停止条件

继续追问“为什么”和“谁”是对的，但不能默认每次分析都能触及根因。材料通常只能支持到某一层机制线索。

应把规则改为：

> 沿现象、用户、行为、机制和能力逐层追问，直到找到有证据支持的最深解释；再往下证据不足时停止，把下一层写成假设或开放问题。

这样既鼓励深化，也避免把合理猜测包装成根因。

## 三、建议的新 Role

建议把 Analysis 的 Role 改写为：

> 你负责把参考材料想明白。先广泛发现可能重要的现象、矛盾、解释和假设，再通过比较、追问、证据核对和反证检验，收敛成一组值得进入汇报的发现与观点。你的产出既不能停留在材料摘要，也不需要提前决定报告的核心主线、论证顺序或章节结构。

Analysis 需要回答五个自然问题：

1. 材料里有哪些真正重要或反常识的发现？
2. 多个材料放在一起，能够形成什么新的认识？
3. 为什么会出现这些现象，还有哪些可能解释？
4. 这些发现改变了我们对问题的什么理解？
5. 哪些观点比较确定，哪些仍然存在冲突、边界或待验证假设？

## 四、建议的核心原则

### 原则一：先充分发现，再开始收敛

不要拿到材料后立即寻找唯一答案。先扫描事实、趋势、差异、异常、原话、冲突和缺口，形成候选发现；再判断哪些只是信息，哪些能够发展为观点。

分析框架是搜索地图，不是结论模板。框架之外出现强信号时，应允许调整原有分解。

### 原则二：从观察走到观点

Analysis 不能停在“数据显示了什么”。一个成熟观点至少需要完成：

```text
观察到了什么
→ 可以怎样解释
→ 多项观察合起来说明什么
→ 改变了什么认识
```

其中，观察必须忠于材料；解释和综合必须标明证据强度与边界。

### 原则三：通过比较和追问深化

根据材料类型灵活使用：

- 时间前后比较；
- 用户或场景分群；
- 竞品和基准对照；
- 总量指标的结构分解；
- 定量现象与定性解释互证；
- 异常值和反例检查；
- 对现象继续追问“为什么”“谁”“在什么条件下”。

追问到证据边界就停止，不为了完整而虚构根因。

### 原则四：保留竞争性解释

观点收敛不是只留下最顺眼的解释。对于同一现象，应保留：

- 当前最有支持的解释；
- 仍然合理的替代解释；
- 会削弱主判断的反证和边界；
- 需要进一步验证的假设。

Analysis 可以判断哪个解释更有证据，但不应为了形成漂亮故事而消除真实的不确定性。

### 原则五：收敛成观点池，而不是故事线

最终结果应是一组经过筛选的关键发现、综合观点和可能方向。它们之间可以存在支撑、解释、冲突或前提关系，但不需要被排成唯一叙事。

Analysis 可以指出：

- 哪些观点证据最强；
- 哪些观点最有洞察；
- 哪些发现可以共同形成一个更高层判断；
- 哪些方向互相冲突或存在取舍；
- 哪些内容只能作为假设。

Analysis 不决定：

- 哪个观点成为整份报告的 apex；
- 汇报先讲什么、后讲什么；
- Executive Summary 如何表述；
- 哪些观点进入正文、附录或被删除；
- 章节和内容单元如何安排。

这些是 Storyline 的工作。

## 五、建议的 Workflow

### 1. 理解任务，但暂不预设结论

明确分析主题、需要支持的决策和材料范围。区分用户希望证明的观点与材料已经支持的事实。

### 2. 浏览材料并发散候选发现

先建立材料全貌，记录：

- 显著事实和趋势；
- 分群、阶段或竞品差异；
- 异常值和反常现象；
- 定性材料中反复出现的语言和机制线索；
- 不同来源之间的一致、互补和冲突；
- 值得验证的假设。

这一阶段允许候选内容冗余，不急于组织成故事。

### 3. 搭建或修正分析框架

用问题分解、指标拆解、用户—行为—机制链等方式检查候选发现是否覆盖了问题的重要方面。框架用于发现遗漏和建立比较关系，不用于决定报告章节。

如果材料中的强信号超出初始框架，应修正框架，而不是舍弃信号。

### 4. 验证、深化并形成观点

对候选发现进行比较、追问和反证：

- 核对证据与口径；
- 区分观察、解释和推断；
- 检查异常值与替代解释；
- 尝试回答“为什么”“谁”“在什么条件下”；
- 将多项相互支持的发现综合为更高层观点；
- 在证据不足处停止并保留假设。

### 5. 筛选和收敛观点池

删除重复、琐碎、与问题无关或无法支持的内容。保留：

- 关键发现；
- 对关键发现的解释；
- 由多个发现综合出的观点；
- 重要反例和边界；
- 互相竞争的方向；
- 尚未解决但会影响决策的问题。

最后检查观点池是否既有足够广度，又已经去掉材料噪声。

## 六、与 Storyline 的衔接方式

Analysis 交付给 Storyline 的内容应让 Storyline 能够直接回答三个问题：

1. 有哪些值得讲的观点？
2. 每个观点为什么成立，最多可以讲到什么程度？
3. 这些观点之间有哪些支撑、解释、冲突或取舍关系？

Storyline 在此基础上完成第二次收敛：

```text
观点池
  → 根据受众和汇报目标选择核心主张
  → 选择支撑观点并决定舍弃内容
  → 建立递进、转折或因果论证
  → 形成 One-line Story、Executive Summary 和报告骨架
```

两者的边界可以概括为：

| Analysis | Storyline |
|---|---|
| 从材料中发现和收敛观点 | 从观点中选择和收敛故事线 |
| 判断观点是否成立 | 判断观点如何服务这次汇报 |
| 建立内容关系 | 建立论证顺序 |
| 保留多个合理解释和方向 | 选择唯一核心主线 |
| 追求想得充分、准确、有洞察 | 追求讲得聚焦、清楚、有说服力 |

## 七、对现有 Skill 的具体修改建议

### 1. Role

将“分析引擎、形成可追溯判断”改为“把材料想明白，通过发散、验证和收敛形成观点池”，直接写清与 Storyline 的二次收敛关系。

### 2. 原则一：结构化分析

- 保留问题分解、结构拆解和避免按材料目录平铺。
- 将“金字塔与 MECE”改成“分析框架与覆盖检查”。
- 强调框架是可修正的搜索地图，不是预设结论或报告结构。
- 不要求所有 findings 提前形成唯一的主—子—支撑层级。

### 3. 原则二：判断淬炼

- 保留 observation → inference → statement → so what 的推理链。
- 将“每个 statement 必须有张力”调整为“综合观点应体现重要差异、反常识或决策张力”。
- 将“追问到根因”调整为“追问到证据能够支持的最深一层”。

### 4. 原则三：筛选与深化

- 改成“先按信息价值发现，再按汇报价值收敛”的两轮筛选。
- 避免因为暂时不可行动而删除解释性、风险性或边界性发现。
- 保留空间、收益、可行性和替代路径，作为最终观点优先级的判断维度。

### 5. 原则五：分析边界

- 把最终产出明确为“观点池”，而不是“最终结论”或“观点金字塔”。
- 允许表达多个综合观点和候选方向。
- 明确禁止选择 apex、设计论证顺序、决定正文与附录、撰写 ES 或章节。

### 6. Workflow

- 在“搭框架”和“形成 Findings”之间增加显式的候选发现发散阶段。
- 把“形成 Findings”改成“验证、深化并形成观点”。
- 把“综合视角”改成“筛选并收敛观点池”。
- 自审增加两个问题：
  - 是否因为初始框架遗漏了框架外的重要信号？
  - 是否已经形成观点，但尚未越界形成唯一故事线？

### 7. Output 描述

在不讨论具体 schema 的前提下，先把语义改成：

- 关键发现；
- 解释与机制线索；
- 综合观点；
- 竞争性解释和反证；
- 观点之间的关系与决策张力；
- 证据强度、边界和开放问题。

避免把 Analysis 的输出描述成最终推荐、唯一观点层级或报告骨架。

## 八、建议的迭代顺序

### 第一轮：只改定位和 SOP

先修改 Role、核心原则、Workflow 和 Output 的语义描述，不改 schema。目标是验证新版 skill 能否稳定做到：

- 比现有版本发现更多真正重要的信号；
- 不停留在材料复述；
- 能把多个发现综合成观点；
- 保留反证与竞争解释；
- 不提前写成 Storyline。

### 第二轮：用真实案例对比

选择三类案例：

1. 定量材料较完整、存在明确结构分解的案例；
2. 定量和访谈混合、需要从现象追到机制的案例；
3. 材料互相冲突或证据不足、无法形成单一答案的案例。

重点进行人工比较：

- 观点发现的广度；
- 观点的洞察深度；
- 从证据到观点的推导质量；
- 对异常、反证和替代解释的处理；
- 对 Storyline 的可用性；
- 是否出现提前设计故事线的越界。

### 第三轮：再判断是否需要调整契约

只有当真实案例反复表明现有产物无法承载“观点池”时，再讨论 schema、rubrics 和交接字段。这样可以先验证认知模型，再固化实现，避免为尚未稳定的工作方法过早设计接口。
