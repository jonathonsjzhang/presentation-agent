# 汇报助手用户指南

> 适用版本：1.7+
> 适用平台：WorkBuddy / Codex / Claude Code

---

## 一、这是什么

汇报助手是一个嵌入在 AI Agent 终端中的汇报材料生成工具，聚焦互联网战略分析场景。用户以自然语言提出汇报需求，全程由 AI Agent 代为调度工具链完成 7 个阶段的逐步生成，各阶段产出均可审阅、修正、确认后再推进，最终交付格式为 PPT、HTML 或文档。

**用户无需执行任何命令行、Python 脚本或配置。** 所有安装、初始化、调度和记忆沉淀均由 AI Agent 在后台完成。

---

## 二、支持哪些平台

汇报助手以 GitHub 仓库形式分发。你需要在以下任一 AI Agent 终端中使用：

| 平台 | 安装方式 | 完成时长 |
|---|---|---|
| **WorkBuddy** | 将仓库地址交给 Agent，由其自动 clone 安装 | 约 2 分钟 |
| **Codex** | 同上 | 约 2 分钟 |
| **Claude Code** | 同上 | 约 2 分钟 |

本节依次说明三种平台的安装与启动方式。

### 2.1 在 WorkBuddy 中使用

**安装：** 在对话中说：

> "帮我安装汇报助手，GitHub 地址是 [你的仓库地址]"

Agent 会依次完成：clone 仓库 → 初始化工作区 → 自检确认可用。完成后回复"安装完成，一切正常"即可使用。

**使用：** 直接提出汇报需求，例如：

> "帮我写一份面向集团高管的云计算行业竞争格局汇报 PPT"

Agent 会自动识别并启动汇报流水线。

**更新：** 当工具发布新版本时，在对话中说：

> "更新汇报助手"

Agent 会执行 `git pull` 拉取最新代码并重新初始化工作区。你的历史数据和反馈不会丢失。

### 2.2 在 Codex 中使用

**安装：** 将仓库地址配置为环境变量（一次性），或直接在对话中提供：

> "请从这个地址安装汇报助手：[你的仓库地址]"

Agent 将完成后续全部安装步骤。

**使用与更新：** 与 WorkBuddy 相同，直接以自然语言提出汇报需求或更新指令。

### 2.3 在 Claude Code 中使用

**安装：** 将 `templates/host_adapters/claude-code/SKILL.md` 放入 Claude Code 的 skill 目录。首次对话中说：

> "安装汇报助手，仓库地址是 [你的仓库地址]"

**使用与更新：** 与上述平台完全一致。

## 三、第一次使用：你会看到什么

以下以一个真实对话为例，展示完整交互过程。

### 3.1 提出需求

第一步是告诉 Agent 你要做什么汇报。Agent 会先与你确认三个必填字段：

| 字段 | 含义 | 示例 |
|---|---|---|
| 汇报主题 (topic) | 你想汇报什么 | "2025 年全球云基础设施市场格局分析" |
| 汇报对象 (audience) | 向谁汇报 | "集团战略投资委员会" |
| 希望支撑的决策 (decision_goal) | 汇报要达到什么目的 | "决定是否增加海外节点投入" |

此外，你还可以补充（非必填）：

| 字段 | 含义 |
|---|---|
| 汇报类型 | `deep_dive`（深度分析，默认）或 `quick_sync`（快速同步） |
| 输出格式 | `ppt`（默认）/ `document` / `html` |
| 背景资料 | 已有的研究结论、数据文件、竞品分析报告的路径或内容 |
| 约束条件 | 页数上限、保密要求、口径限制等 |
| 真实意图 | 一句话描述这次汇报最核心想达到什么效果 |

**示例对话：**

> **你：** 帮我写一份汇报，主题是 AI 对互联网广告行业的重塑。汇报对象是 CEO 和 COO，要支撑"明年是否加大 AI 广告产品投入"这个决策。我有一些竞品数据可以先发给你。

> **Agent：** 好的，我已经记下主题、对象和决策目标三项关键信息。请把竞品数据发给我，另外补充几个问题：
> 1. 汇报页数大概多少？
> 2. 是否有特定的口径或保密要求？
> 3. 你对"AI 广告产品"最想强调的是什么？

> **你：** 10 页左右。不要提具体客户名字。最想强调的是我们的数据积累优势是护城河。

### 3.2 开始生成

确认 brief 后，Agent 会告知你"汇报流水线已启动"，然后进入第 1 阶段。此后，每个阶段结束时 Agent 会展示产出摘要并等待你的意见。

## 四、7 个阶段：每个阶段做什么、产出什么

回报助手按以下顺序执行 7 个阶段。每个阶段完成后，Agent 会暂停等待你的确认。

### 阶段 1：任务定位

**做什么：** 将你提供的汇报主题、对象、素材和约束条件收敛为一份结构化的执行 brief，同时初始化全局配置（受众、目标 action、页数上限等）。

**你会看到：** 一份整理后的汇报概要，包含 Agent 对任务的理解。

**你需要做什么：** 核对概要是否准确，如有偏差请直接指出。

---

### 阶段 2：核心论点提炼

**做什么：** 基于你提供的素材和背景，形成 Executive Summary——包括核心结论、关键论点、支撑论据和预期行动。

**你会看到：** 一段塔尖式总结（核心结论在前），以及各论点与证据的对齐关系。

**你需要做什么：** 确认结论方向是否正确、论据是否有误或遗漏。

---

### 阶段 3：Storyline 设计

**做什么：** 把 Executive Summary 拆解为逐页的叙事结构——每页的标题、要回答的关键问题、"So What"和预期放置的证据素材。

**你会看到：** 一份完整的页级大纲，含每页标题和逻辑衔接。

**你需要做什么：** 确认叙事逻辑是否流畅、页数是否合适、重点是否突出。

---

### 阶段 4：单页内容填充

**做什么：** 为每一页生成具体内容简稿——页面结构、图表方案、证据摆放位置及缺口标记。

**你会看到：** 每页的内容框架，包括图表建议和当前缺什么数据。

**你需要做什么：** 逐页审阅内容方向和证据充分性，补充缺失材料。

---

### 阶段 5：格式渲染

**做什么：** 将前述内容框架渲染为最终格式的正式材料（PPT / HTML / 文档），保持结论与证据不变形。

**你会看到：** 可打开的最终文件或可预览的 HTML 页面。

**你需要做什么：** 审阅版式和视觉呈现，关注格式细节。

---

### 阶段 6：Q&A 梳理

**做什么：** 基于正式材料，预判汇报对象可能的提问、挑战角度，并准备回答策略。标注需要汇报人自行补充的信息。

**你会看到：** 一份按可能性排序的 Q&A 列表，含建议的回答方向。

**你需要做什么：** 补充你了解的敏感问题和回答口径。

---

### 阶段 7：逐字稿

**做什么：** 基于正式材料和 Q&A 包生成完整汇报逐字稿，逐页对应材料，默认控制在 30 分钟以内。

**你会看到：** 一份可逐句跟读的演讲文稿。

**你需要做什么：** 调整语气、节奏和个人表达习惯。

---

### 关于记忆：它会在使用中越来越懂你

每次你在阶段审阅中给出的修改意见（即使是随口一句点评），Agent 都会自动记录到对应阶段的记忆中。下次同样的阶段、同样的汇报类型，工具会主动参考你过去的偏好来调整产出方向。

例如：你第一次说"结论太委婉，我的汇报对象习惯开门见山"，第二次做类似的汇报时，结论就会自然地前置。**你不需要填任何表格——说一句话就够了。**

## 五、使用前你需要准备什么

汇报助手不会凭空生成内容。它需要一个**素材包**，可以是一份文档、一堆数据、几条结论、或者几段你自己写的文字。

| 必备项 | 最简形式 | 更好的形式 |
|---|---|---|
| 汇报主题与对象 | 一句话描述 | 含背景和决策目标 |
| 素材 | 口头描述 | 一份文档文件或几点结论摘要 |

**一个常见误区：** 工具不要求你提供完整的分析报告（那是工具要产出的）。工具需要的是你的**已知信息和初步判断**——哪怕只是几句话。

## 六、常用说法速查

以下是你可能用到的话，Agent 会自动识别并执行相应操作：

| 你想做的事 | 你可以这样说 |
|---|---|
| 安装工具 | "帮我安装汇报助手" |
| 更新工具 | "更新汇报助手" |
| 发起汇报 | "帮我写一份关于 XX 的汇报 PPT，给 YY 看" |
| 查看进度 | "现在汇报进行到哪个阶段了" |
| 提出修改意见 | "第二页结论不够尖锐，改成…""数据不准，我用最新值替换如下…" |
| 认可某部分 | "这一页很好，记下来以后都这样" |
| 暂停 | "先停一下，我晚上再继续" |
| 重新生成某一页 | "第五页重做，方向改成…" |

## 七、文件放在哪里

| 内容 | 位置 |
|---|---|
| 汇报工具程序 | `~/PresentationAgent/repo/` |
| 你所有的汇报数据、历史记录和记忆 | `~/PresentationAgent/workspaces/default/` |
| 每次汇报的最终产物 | `~/PresentationAgent/workspaces/default/runs/<run_id>/output/` |

更新汇报工具时，只更新程序文件（`repo/`），你的数据（`workspaces/`）不会被覆盖或删除。

## 八、常见问题

### Q: 需要我会 Python / Git 吗？
不需要。你对 Agent 说话，Agent 完成所有操作。本节提到的命令行全部由 Agent 执行，你不必接触。

### Q: 需要自己申请模型 API Key 吗？
不需要。汇报助手本身不调用任何模型——生成内容、审阅质量全部由你当前使用的 AI Agent（WorkBuddy / Codex / Claude Code）的模型完成。你只需用你已有的 AI Agent 账号即可。

### Q: 如果中途断了怎么办？
断点续传是支持的。下次回到同一 Agent 对话时，说"继续上次的汇报"，Agent 会从断点处恢复。所有中间产物都保存在 workspace 中，不会丢失。

### Q: 可以同时做多份汇报吗？
可以。每次发起新汇报会生成新的 `run_id`，互不干扰。

### Q: 怎么知道反馈被记录了？
Agent 记录你的反馈后，会回复一句"我已把这条反馈记入本阶段 memory"。你可以在对话中随时询问"我之前的反馈记录还有效吗"来查看。

### Q: 可以删掉旧的汇报数据吗？
workspace 中的 runs 目录可按 `run_id` 子目录手动删除。不建议直接删除整个 workspace——其中包含你积累的记忆数据，删除后工具需要从零学习你的偏好。

## 九、快速上手三步

1. **安装**——告诉 Agent："帮我安装汇报助手，地址是 [仓库地址]"
2. **说需求**——"帮我做一份关于 XX 的汇报 PPT，给 YY 看，支撑 ZZ 决策"
3. **确认推进**——每个阶段看产出、提意见、说"继续"，7 步走完即可拿到最终材料
