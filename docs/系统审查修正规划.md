## Phase 2：主路径收敛

### 2.1 统一执行入口：让 `launch.py` 支持 Manager 路径

**问题**：`launch_report()` 自称 host wrapper 的主入口函数，但内部只走 `Pipeline`（Legacy 直接流水线）。Manager 编排路径仅能通过 CLI `report start` 访问。

**行动**：

1. 在 `launch_report()` 中增加 `use_manager: bool = True` 参数（默认走 Manager）
2. Manager 路径下：调用 `ManagerOrchestrator` 初始化 run，返回 Manager planning 指令给宿主
3. Legacy 路径保留但标记 `deprecated`，仅用于向后兼容和调试

**影响文件**：
```
presentation_agent/launch.py         # 主改动
skills/report_builder/SKILL.md       # 更新调度协议说明
```

**launch.py 改动要点**：
```python
def launch_report(
    brief: BriefInput,
    root: Union[str, Path] = ".",
    use_manager: bool = True,        # 新增
    provider: Optional[str] = None,   # Manager 路径不需要 provider
    auto: bool = False,
    out: Optional[Union[str, Path]] = None,
    init_only: bool = False,
) -> dict[str, Any]:
    # ... brief 标准化逻辑不变 ...

    if use_manager:
        # 走 Manager 编排路径
        orchestrator = ManagerOrchestrator(...)
        result = orchestrator.initialize_run(brief_path)
        return result
    else:
        # 走 Legacy Pipeline（deprecated）
        pipeline = Pipeline(root_path, provider_override=provider)
        return pipeline.run(brief_path, run_dir=out_root, auto=auto)
```

**验证**：
- 通过 `skills/report_builder/SKILL.md` 发起汇报，确认走 Manager 路径
- `report next` 返回 Manager planning 指令
- 现有测试 `test_pipeline.py` 仍然通过（Legacy 路径未被删除）

### 2.2 在 StepRunner 文档中标注 StopChecker 差异

**行动**：

1. 在 `step.py` 第 100 行添加注释：

```python
# 宿主自执行模式下不创建独立的 LLM checker。

# LLM sanity check 由宿主在 review 阶段通过 _compose_review_instruction 完成。

```

2. 在 README 第 4.3 节「Review / Stop Check 模块」末尾补充一段说明 Manager 路径的 stop check 机制差异。

**影响文件**：
```

```

## Phase 5：能力补齐

### 5.1 为 Manager acceptance 阶段增加 memory scan

**问题**：Worker 侧通过 `StepRunner._commit_review()` → `memory.record_objections()` 完成记忆回写闭环，但 Manager 在 acceptance 阶段没有对自身决策做 memory scan。

**行动**：

1. 在 `ManagerOrchestrator.commit_manager()` 的 acceptance 分支中，增加对 Manager 自身 memory 的扫描：

```python
# 伪代码
if phase == "acceptance":
    # 扫描 Manager memory 中与「验收」「返工」相关的条目
    memory_alerts = manager_runtime.memory.scan(
        dimensions=["验收", "返工", "跨阶段一致性"],
        context=acceptance_report
    )
    # 如果有命中，作为提醒附加到 acceptance 结果中
    if memory_alerts:
        decision["memory_alerts"] = memory_alerts
```

2. 将 Manager 的 acceptance 决策结果写入 learning_log：

```python
manager_runtime.memory.record_objections(
    run_id,
    acceptance_objections
)
```

**影响文件**：
```
presentation_agent/manager.py         # ManagerOrchestrator.commit_manager()
```

**验证**：运行一次完整 Manager 编排流程，确认 `data/agents/manager/learning_log.jsonl` 中有 acceptance 阶段的记录写入。
